# 项目管理工具 — 使用指南

> 版本：1.0.0  
> 技术栈：Electron 28 + Vue 3 + Vite + TypeScript

---

## 一、项目概述

**项目管理工具** 是一款 Windows 桌面客户端，用于集中管理本地开发项目与常用工具。它支持以项目为单位组织代码目录、快速启动编辑器、全局搜索项目/代码/工具，并提供深色/浅色主题切换。

核心能力：

- **项目管理**：手动添加 / 批量扫描导入项目，支持自定义分组、星标、标签、排序
- **代码模块识别**：自动扫描 Java / Vue / React / Go / Python / .NET 等项目的多模块结构
- **编辑器集成**：自动检测 VSCode / IntelliJ IDEA / Trae CN / Qoder 等编辑器，一键打开
- **工具面板**：添加本地可执行文件、目录或命令行工具，支持图标扫描与网格化展示
- **全局搜索**：`Alt+Space` 全局呼出独立搜索窗口，搜索项目/文件/代码模块/工具
- **体验优化**：系统托盘驻留、无边框窗口、深色/浅色/跟随系统三种主题

## 三、安装与启动

### 3.1 绿色版（推荐）

将打包后的文件夹复制到任意位置，双击 `项目管理工具.exe` 启动。

### 3.2 开发者模式

```bash
# 安装依赖
npm install

# 开发调试
npm run dev

# 构建 EXE
npm run build
```

> 数据文件存储在应用同级 `data/` 目录下（`projects.json`、`settings.json`、`groups.json`、`tools.json`），拷贝整个文件夹即可迁移。

## 五、功能模块详解

### 5.1 项目管理

#### 5.1.1 新建项目

1. 点击右上角 **「新建项目」** 按钮
2. 填写以下内容：

| 字段 | 说明 |
|------|------|
| 项目名称（必填） | 自定义项目名称 |
| 项目描述 | 简短说明项目用途 |
| 项目根目录（必填） | 选择项目的根文件夹路径 |
| 代码模块 | 手动添加子模块目录，或点击「自动扫描代码」智能识别 |
| 文档目录 | 关联文档存放位置 |
| 标签 | 按回车添加分类标签 |
| 所属分组 | 归入已有分组或保持"未分组" |

3. 点击 **「创建」** 完成

#### 5.1.2 自动扫描代码模块

在新建/编辑项目对话框中选择根目录后，点击 **「自动扫描代码」** 按钮，工具会自动：

- 递归扫描（最大深度 3 层）项目目录
- 识别 Java（`pom.xml`、`build.gradle`）、Vue/React（`package.json`）、Go（`go.mod`）、Python（`requirements.txt`/`setup.py`）、.NET（`.csproj`/`.sln`）等代码模块
- 自动命名模块（后端/前端/API 等）并标注语言与框架

#### 5.1.3 项目列表操作

在项目列表页面，每个项目卡片提供以下快捷操作：

| 操作 | 方式 |
|------|------|
| 打开项目详情 | 单击项目卡片 |
| 用编辑器打开 | 点击项目卡片右侧编辑器图标（自动检测已安装编辑器） |
| 打开文件夹 | 点击文件夹图标，在资源管理器中打开 |
| 星标/取消星标 | 点击星标按钮，星标项目左侧高亮显示 |
| 编辑项目 | 点击编辑按钮修改项目信息 |
| 删除项目 | 点击删除按钮（仅移除管理记录，不删除文件） |
| 右键菜单 | 右键项目卡片，弹出完整操作菜单 |
| 置顶模块快捷打开 | 点击项目底部的置顶模块标签直接打开 |

#### 5.1.4 分组管理

- **新建分组**：点击「新建分组」→ 输入名称 → 创建
- **重命名分组**：悬停分组标题 → 点击编辑图标
- **删除分组**：点击删除图标（分组内的项目自动归入"未分组"）
- **折叠/展开**：点击分组标题可折叠或展开该分组
- **内置分组**：
  - **星标项目**：自动聚合所有已星标项目
  - **未分组**：聚合所有未归入自定义分组的项目

#### 5.1.5 拖拽排序与跨组移动

- **同组排序**：拖拽项目卡片手柄（悬停可见）调整项目在组内的顺序
- **跨组移动**：将项目拖拽到另一分组即可更改归属
- **排序方式**：右上角下拉菜单支持「手动排序」「最近更新」「按名称」「创建时间」

#### 5.1.6 批量操作

- **进入多选模式**：点击项目卡片的复选框，或使用 `Ctrl+点击` 选择多个项目
- **全选**：`Ctrl+A`
- **批量操作**：出现蓝色的批处理工具栏，支持：
  - 批量星标 / 取消星标
  - 批量移至分组
  - 批量删除
- **退出多选**：按 `Esc` 键

#### 5.1.7 项目详情页

点击项目进入详情页，展示：

- **代码模块列表**：每张模块卡片显示名称、路径、语言/框架信息
  - 支持**置顶/取消置顶**模块（星标按钮），置顶模块在列表和左侧导航栏中优先展示
  - 支持**切换编辑器**打开（下拉菜单选择 VSCode / IDEA / Trae CN / Qoder）
- **文档目录**：显示关联的文档路径，一键打开
- **项目信息**：路径、描述、创建时间、更新时间、标签
- **快捷操作**：打开项目文件夹、用首选编辑器打开

### 5.2 工具面板

#### 5.2.1 工具类型

支持三种类型的工具：

| 类型 | 说明 | 示例 |
|------|------|------|
| 可执行文件 | 本地 `.exe` / `.bat` / `.cmd` / `.ps1` 程序 | `D:\Tools\Everything.exe` |
| 目录 | 文件夹路径，点击在资源管理器中打开 | `D:\Documents\设计稿` |
| 命令 | 自定义命令行指令 | `npm run dev`、`python script.py` |

#### 5.2.2 添加工具

1. 进入「工具」页面，切换到 **「管理」** 视图
2. 点击 **「添加工具」**
3. 选择类型，填写名称、路径（可执行文件/目录/命令）
4. 可选：启动参数、默认工作区、描述、标签
5. 点击「添加」

#### 5.2.3 扫描导入

点击 **「扫描导入」** → 选择目录 → 开始扫描，工具会：

- 自动扫描目录下的所有 `.exe` / `.bat` / `.cmd` / `.ps1` 文件
- 自动扫描一级子目录
- 导入为新工具（已存在的自动跳过）

#### 5.2.4 双视图模式

- **管理模式**：列表视图，支持编辑、删除、打开
- **面板模式**：网格布局，类似 Launchpad 风格，适合快速点击启动
  - 支持**扫描图标**：自动查找工具目录下的 `.ico` / `.png` / `.svg` 图标文件并展示
  - 未找到图标时显示文字首字母，颜色自动分配

### 5.3 全局搜索

#### 5.3.1 两种搜索方式

| 方式 | 快捷键 | 说明 |
|------|--------|------|
| 全局搜索窗口 | `Alt + Space` | 独立的置顶搜索窗口，在任何应用上方显示 |
| 应用内搜索 | `Ctrl + K` | 应用内部的半透明搜索覆盖层 |

#### 5.3.2 搜索范围

全局搜索会检索：

- **项目**：项目名称、描述、标签
- **文件**：项目目录下的文件名（缓存 5 分钟）
- **代码模块**：模块名称
- **工具**：工具名称

结果按相关度评分排序，最多展示 20 条。

#### 5.3.3 使用搜索覆盖层（Ctrl+K）

1. 按下 `Ctrl+K` 打开搜索覆盖层
2. 输入关键词，实时出结果（300ms 防抖）
3. 键盘 `↑` `↓` 导航，`Enter` 跳转，`Esc` 关闭
4. 无输入时展示：**最近项目**（最近访问 5 个）+ **快捷操作**（新建项目、打开设置、刷新列表）

#### 5.3.4 全局搜索窗口（Alt+Space）

按 `Alt+Space` 呼出独立搜索窗口（680×500，透明置顶），在搜索结果中：

- 点击「项目」→ 打开项目详情
- 点击「文件」→ 在资源管理器中打开
- 点击「工具」→ 直接启动工具
- 点击「代码模块」→ 跳转到对应项目

- **拖拽排序**：调整编辑器在快捷按钮中的显示顺序
- **删除**：移除不再需要的自定义编辑器

## 七、常见问题

### Q1：删除项目会删除项目文件吗？

**不会。** 删除操作仅从管理列表中移除该项目记录，不会对磁盘上的实际文件做任何改动。

### Q2：为什么有些编辑器检测不到？

编辑器检测依赖注册表或默认安装路径。如果编辑器安装在非标准路径，可以在「设置 → 编辑器管理」中手动编辑路径，或添加为自定义编辑器。

### Q3：代码模块扫描不准确怎么办？

自动扫描基于常见构建配置文件进行识别。如果结果不准确，可以在项目编辑对话框中**手动添加或删除代码模块**，手动指定模块标签、路径和关联的编辑器。

### Q4：工具展示模式下的图标没有显示？

点击面板模式下的 **「扫描图标」** 按钮，工具会自动在可执行文件所在目录查找 `.ico`、`.png`、`.svg` 图标文件。如果目录下没有图标文件，则显示文字首字母。

### Q5：如何迁移到另一台电脑？

- **绿色版**：直接复制整个应用文件夹到新电脑即可，`data/` 目录包含所有数据
- **若只迁移数据**：复制 `data/` 文件夹中的所有 `.json` 文件到新电脑对应位置

### Q6：全局搜索窗口和主窗口的关系？

它们是两个独立的 Electron 窗口。全局搜索窗口（`Alt+Space`）是独立的、透明置顶的小窗口；应用内搜索（`Ctrl+K`）是覆盖在主窗口上的半透明遮罩层。两者功能相同，使用场景不同。

### Q7：如何彻底退出程序？

右键点击系统托盘中的图标 → 选择 **「退出」**。直接关闭窗口只会最小化到托盘。
